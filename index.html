Digital Accessibility:
A Starting Point for Faculty

Prepared by Ka-eul Yoo · Global & International Studies, UCI

GIS Faculty Meeting · March 11, 2026

Compliance Deadline April 24,
2026
With limited exceptions, digital materials made available to students online — including Canvas content, PDFs, slides, videos, and documents — should be accessible. This is a starting point: what I've gathered from several workshops and resources. It's not complete, and it won't be perfect, but it's a baseline we can build on together!

Note: This guide reflects my own synthesis of available resources and may not capture every detail — for official policy, refer to UCI Accessibility Hub.
The Legal Context
The Federal Law
ADA Title II
The Americans with Disabilities Act requires public universities to make all digital content accessible. A 2024 DOJ final rule clarified specific accessibility requirements for the web content and digital materials of public entities — with significant implications for online course materials, documents, and instructional content.
The Standard
WCAG 2.1 Level AA
Web Content Accessibility Guidelines — the international benchmark for accessible digital content. Covers Canvas materials: documents, slides, videos, and websites. Note: WCAG 2.2 is the current version, but the DOJ and UC policy currently reference 2.1 AA as the required baseline.
UCI's Internal Policy
UC IMT-1300
The long-standing UC system-wide policy requiring all UC institutions to meet WCAG 2.1 AA. The 2026 DOJ deadline is now accelerating compliance efforts across all campuses.
YuJa Panorama — Check What's Already in Canvas

What is YuJa Panorama?

YuJa Panorama is an accessibility platform integrated with Canvas that automatically scans your course files and flags accessibility issues. It displays color-coded indicators next to files and generates reports that help you fix problems — some right inside Canvas. It also automatically generates alternative formats for students (audio, HTML, tagged PDF), improving access without requiring additional work from you.

Green = Accessible. No action needed.
Yellow = Some issues. Fix after red items.
Red = Significant issues. Fix these first.
How to Fix Issues in YuJa Panorama
  1. 1
    Open your Canvas course Look for the colored circles next to your files. Start with the red ones.
  2. 2
    Click → View → Fix Issue YuJa Panorama walks you through each problem one by one, right inside Canvas.
  3. 3
    Fix simple issues in YuJa Panorama Alt text for images, slide titles, link text, color contrast — all fixable in-place.
  4. 4
    Complex issues? Contact the DASH (Digital Accessibility Support & Help) Team Submit a request form; they handle the harder stuff — complex PDFs, scanned documents, OCR.
  5. 5
    100% green = YuJa Panorama has no further flags YuJa Panorama is your final check. Green across the board means no further flags.
Most Common Issues to Look For
  • Images without Alt Text
    Add a brief description of what the image shows. Decorative images? Mark as “decorative” — screen readers will skip them.
  • Missing Slide Titles
    Every slide needs a title (can be hidden) so screen readers can navigate between them.
  • Vague Link Text
    “Click here” → use descriptive text like “Download Week 3 Readings.”
  • PDFs without Tags
    Use Adobe Pro's “Prepare for Accessibility” auto-tag feature, then re-upload to Canvas.
  • Videos without Captions
    Auto-captions can be a starting point, but always review for accuracy. For lecture recordings, use YuJa's caption editor to review and correct captions.
  • Low Color Contrast
    YuJa Panorama flags text that's hard to read and lets you pick an accessible color right in Canvas.
Lower-Priority & Potential Exceptions

Potential exceptions — context-dependent, check campus guidance

  • Clearly archived content kept only for reference or recordkeeping (not actively used for teaching)
  • Files not uploaded to Canvas (shown in class only)
  • Some unaffiliated third-party content — but not content provided through a university contract or licensing arrangement
  • Content published before the April 24, 2026 deadline that is no longer active
  • Individualized, password-protected documents specific to a single person

Important Past-quarter materials still accessible to students in Canvas are considered active, not archived — and are in scope. If you don't want them in scope, remove them from Canvas or restrict access.

A Few Things Worth Knowing

Accessibility vs. Accommodation: Accessibility is proactive course design for everyone; accommodations are individualized adjustments for specific students. Both matter — but they're different obligations.

The expectation is not instant perfection. What matters is taking reasonable, documented steps to improve accessibility — especially for materials actively in use. Schools that faced legal action historically did so because they refused to act, not because they were imperfect.

How to Check Accessibility in Your Tools
YuJa Panorama — Canvas
  • Open your Canvas course
  • Look for colored circles next to each file
  • Click a red circle → View
  • Click Fix Issue on each item
  • For images: add alt text or mark as Decorative
  • For slides: add or set a Slide Title
  • For links: replace “click here” with descriptive text
  • When everything is green → done

Important: These categories are narrow and context-dependent. If the material is actively used for teaching or student access, treat it as in scope.

📊 PowerPoint
  • Open your .pptx file
  • Click Review tab in the top menu
  • Click Check Accessibility
  • Panel opens on the right with a list of issues
  • Click each issue → it takes you directly to the problem
  • Right-click any image → Edit Alt Text
  • Add a slide title via View → Outline View
  • Fix Reading Order: Review → Check Accessibility → Reading Order Pane
  • Re-upload to Canvas when done
📄 Word & Adobe PDF
  • Word: Review tab → Check Accessibility
  • Use Heading styles (H1, H2) — not just bold text
  • Add alt text to all images (right-click → Edit Alt Text)
  • Use descriptive hyperlink text, not bare URLs
  • Export as PDF: Save As → PDF (preserves tags)
  • Adobe Acrobat Pro: All Tools → Prepare for Accessibility
  • Click Autotag Document — runs automatically
  • Then add alt text to any flagged images
  • Upload the tagged PDF to Canvas
📚 Google Slides
  • No built-in accessibility checker — install the Grackle Slides add-on (free)
  • Extensions → Add-ons → Get add-ons → search Grackle
  • After install: Extensions → Grackle Slides → Launch
  • Add alt text: click image → right-click → Alt text
  • Check reading order: press Tab through each slide
  • Use built-in slide layouts — don't start from a blank slide
  • Auto captions available when presenting in Chrome
🎨 Canva
  • File → Accessibility → Check Design Accessibility
  • Flags typography, color contrast, and missing alt text
  • Right-click image → Alternative text to add descriptions
  • Use the Position tool to set logical reading order (bottom layer = read first)
  • Canva PDFs may still have reading order issues — check in Acrobat Pro after export
  • For complex multi-page documents, consider using Word instead
Font, Color & Design — Quick Reference
Font
Sans-serif fonts
WCAG does not mandate specific fonts or minimum font sizes for digital documents. However, common accessibility guidelines recommend using sans-serif fonts (e.g., Arial, Calibri, Aptos, Helvetica) for readability, avoiding decorative or cursive fonts, and using 12–14pt for Word documents and PDFs and 18–24pt for PowerPoint slides (larger when projected). Headings should be bold and clearly distinct from body text.
Color & Contrast
4.5:1 minimum ratio
Never use color alone to convey meaning — e.g., marking a deadline only in red text. Color alone doesn't carry meaning for everyone — different ways of seeing color, grayscale printing, and low-contrast screens all affect how content reads. Always pair color with bold text or an icon (⚠️). YuJa Panorama and PowerPoint's checker will flag contrast issues automatically.
